# football-lineup-generator

## football-{name of the team}.csv
    This will contain all the players of a team with the player's positions and name
    e.x., : Cristiano Ronaldo;ST
            João Félix;AM,LW,ST
    Players can have multiple positions.
    The valid positions are : 
        For forwards; ST,LW,RW,AM
        For midfielders; MC,MR,ML,MD
        For defenders; DM,DL,DC,DR
        For goalers; G

## lineup
    This will create and manage the lineup.
    The lineup is a table that automatically add a football position in the table.
    The user may choose any valid lineup.
    e.x, : 4-3-3 (note that the goaler is not included)
           4-1-2-3 (for a DM)
           4-2-1-3 (for a AM)
    All the modification to the lineup will happen in this file.

## team
    This will contain a table of player depending on the team selected.
    Will send this table to lineup.lua

## main
    Select the team to modify or use it. There is also an option to add a new team (all the data are contained in a csv).
    Contain a button for the instructions.
    When you choose a team a lineup graphic will appear. Nothing his display until the lineup is valid.
    Once its valid a graphic containhing block of the visible formation will appear and a button generate will appear.
    When button generate appear call lineup.lua and fill the square with player's name.

## The generator
    The lineup is generated with these rules :
    lineup[0][0] will always be the goal keeper. There's always one goaler.
    lineup[1][...] will always be the defenders. only the far left and far right are respectly DL AND DR. Every other defenders are DC. If 3 or less defenders then all willbe DC.
    lineup[2][1] AND rows >= 4 if there is only one player in this then it will be a DM.
    lineup[2][...] if there is more than ten then it will be midfielders. Same rules for defenders, far right and far left are respectly MR and ML. Other are MC if there is 3 midfielders or less then all will be MC.

    if there is a DM then apply the same logic as lineup[2][...] but for [3][...] instead
